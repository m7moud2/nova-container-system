Nova Dashboard Docs

Welcome to the Nova enterprise documentation. From here you can learn how to manage your high-performance serverless deployments directly through the Dashboard UI.

Tip: Everything you can do in this dashboard can also be automated via our REST API using standard bearer token authentication.

Phase 1: Deploying your First Project

Nova is designed to be completely zero-config. Whether you build Locally or Connect via GitHub, deployment is handled under the hood automatically.

1

Create Project

In the Deployments tab, select "New Deployment". Choose a pre-built template (like Next.js or Python Flask) or point Nova to your GitHub repository.

2

Engine Scanner

Once selected, Nova's Intelligent Runtime Scanner will determine exactly what compilation steps your app needs and build a minimal Wasm binary representing your codebase.

3

Edge Propagation

In milliseconds, your newly generated WebAssembly binary is pushed out to 42 Edge locations and is instantly globally available on a secure https://your-project.nova.cloud domain.

Phase 2: Understand Billing

Nova Cloud uses a strict pay-as-you-go model. You never pay for idle resources because Nova scales down to ZERO instantly.

Compute Usage

$0.005 per CPU hour.
Since execution is rapid (ms), your cost drops drastically compared to EC2 instances.

Egress Bandwidth

$0.01 per GB.
The first 50GB of outbound data transfer per month is completely free.

Phase 3: Control & Monitoring

Managing deployments in Production shouldn't be stressful. Your Project details page offers immediate control.

  • Soft Restart: Restarting your container takes < 1ms, resulting in zero visible downtime to your users.
  • Live Analytics: View your response times, memory peaks, and active requests rendered smoothly with Chart.js based metric aggregation.
  • Streaming Logs: View STDOUT and STDERR from your application securely streamed via WebSockets to your browser in real time.

Terminal & CLI Access

Use our CLI for robust local-to-cloud workflows.
