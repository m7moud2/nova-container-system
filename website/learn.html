What is Nova Cloud?

Nova is the next-generation serverless container platform built entirely on WebAssembly (Wasm). It completely revolutionizes how we deploy, run, and scale applications without Linux container overhead.

The Problem Nova Solves

Traditional containers (Docker, Podman) are:

  • Slow to start (2-5 seconds)
  • Heavy on resources (100MB+ overhead)
  • Complex to configure via Kubernetes
  • Platform-dependent (Linux kernel required)

The Nova Solution

Nova uses WebAssembly to provide:

  • 240µs startup (1000x faster)
  • 5MB overhead (20x lighter)
  • Zero config (just run your code)
  • True portability (any OS, any CPU)

Installation

Get Nova running on your local machine in under 60 seconds.

macOS & Linux: Our one-line installer sets everything up via Bash.

Bash
$ curl -sSf https://raw.githubusercontent.com/m7moud2/nova-container-system/main/install.sh | sh

Or, if you prefer building from source, Nova is built in Rust natively:

Bash
$ git clone https://github.com/m7moud2/nova-container-system.git $ cd nova-container-system $ cargo build --release $ sudo cp target/release/nova_container_system /usr/local/bin/nova

Windows Users: Download the pre-compiled executable directly using PowerShell: iwr https://github.com/m7moud2/nova-container-system/releases/latest/download/nova-windows-x86_64.exe -OutFile nova.exe

Quickstart: Your first container

Start your first application in 3 intuitive steps. Nova auto-detects what you're trying to do.

  • 1
    Create Your App:
    $ echo 'print("Hello from Nova!")' > app.py
  • 2
    Run with Nova:
    $ nova run app.py
  • That's It! 🎉

    Nova automatically detected Python and ran your app natively inside a sandboxed Wasm isolate in 240µs.

Edge Networking

Nova's Global Edge Network ensures your application is served from the data center physically closest to your users. When you deploy, your application is replicated across our Anycast network.

  • Automatic Routing: Users in Tokyo are routed to our AP-Northeast instances, while users in London hit EU-West.
  • DDoS Protection: Built-in L4/L7 mitigation drops malicious traffic before it ever reaches your Wasm isolate.
  • Built-in SSL: Every deployment automatically receives a managed TLS 1.3 certificate.

Zero-Trust Isolation

Unlike Docker and Kubernetes which share the host operating system kernel, Nova utilizes WebAssembly's mathematical memory safety guarantees.

Container Escapes are Impossible: Wasm isolates run inside a strictly bounded memory sandbox. They cannot access the host filesystem, network, or other processes unless explicitly permitted via WASI capabilities.

Frameworks: Python & Django

Run your Python applications seamlessly. Nova automatically detects requirements.txt or pyproject.toml.

Python Deployment
$ nova deploy ./my-django-app --framework django ✓ Python environment analyzed ✓ Compiled to Wasm payload (14MB) ✓ Deployed securely

Frameworks: Node.js & Next.js

Full support for modern JavaScript and TypeScript frameworks, including Server-Side Rendering (SSR) in Next.js.

Next.js Deployment
$ nova deploy ./my-next-app --framework nextjs ✓ Next.js build detected ✓ Static assets pushed to Edge CDN ✓ SSR endpoints compiled to Wasm

Frameworks: Rust Native

Rust is a first-class citizen on Nova Cloud. Compile your Rust applications directly to the wasm32-wasi target for unparalleled performance.

Rust Execution
$ cargo build --target wasm32-wasi --release $ nova run target/wasm32-wasi/release/app.wasm ✓ Native Wasm module executed in 240µs

API Request Example
$ curl -X POST https://api.nova.cloud/v1/deploy \ -H "Authorization: Bearer nc_live_12345" \ -H "Content-Type: application/json" \ -d '{"target_id": "proj_xyz", "image_tag": "latest"}'

Architecture details

Traditional containerization requires a hypervisor or OS-level cgroups that carry significant baseline memory and startup overhead.

  • V8 Isolates & Wasmtime: Instead of booting an OS kernel, we configure isolated WebAssembly memory segments.
  • Distributed Mesh: When you deploy, the compiled Wasm binary is immediately replicated across 42 Edge locations around the globe.
  • Zero-Trust Sandboxing: Mathematical memory safety provides absolute protection against container escapes.
